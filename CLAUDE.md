## Project Overview

`fast-that-hash` (binary name: `fth`) is a Rust CLI tool that identifies hashes, keys, credentials, MAC addresses, phone codes, file signatures, and more. It is a Rust port of [Name-That-Hash](https://github.com/HashPals/Name-That-Hash).

## Build & Test Commands

```sh
cargo build                  # dev build
cargo build --release        # release build
cargo test                   # run all tests
cargo test formats           # run tests in a specific module
cargo test test_pwdump       # run a single test by name
cargo run -- '<input>'       # run against input (use single quotes on Linux)
```

A `justfile` is available: `just run '<input>'`, `just build`, `just test`, `just gen`.

## Code Generation

Several source files are auto-generated from Python scripts. **Do not hand-edit these files:**

- `src/hashes.rs` — generated by `gen_rust.py` (reads from `hashes.py` which defines hash prototypes with regex patterns, hashcat/john modes)
- `src/regex_patterns.rs` — generated by `gen_data.py` (reads JSON from `docs/` for regex patterns)
- `src/mac_vendors.rs`, `src/mastercard.rs`, `src/phone_codes.rs`, `src/file_signatures.rs` — generated by `gen_data.py` (reads JSON lookup data from `docs/`)

Regenerate with: `just gen` (runs `python3 gen_rust.py && python3 gen_data.py`)

## Architecture

### Identification Pipeline

`main.rs` orchestrates the pipeline: parse CLI args → resolve input → run identifiers → filter → sort → output.

The core abstraction is the `Identify` trait (`src/identifier.rs`):
```
trait Identify { fn identify(&self, input: &str) -> Vec<Match>; }
```

Seven identifiers run in order via `all_identifiers()`:
1. **FormatIdentifier** — structured formats (pwdump, /etc/shadow lines, DCC dumps) with high confidence (0.9 rarity)
2. **HashIdentifier** — regex-based hash matching using `RegexSet` for fast multi-pattern matching
3. **MastercardIdentifier** — card number BIN prefix lookups
4. **RegexIdentifier** — credential patterns (API keys, SSH keys, CTF flags, URLs, etc.)
5. **MacVendorIdentifier** — MAC address OUI vendor lookups
6. **PhoneCodeIdentifier** — international dialing code lookups
7. **FileSignatureIdentifier** — hex magic byte identification

### Key Modules

- `src/formats.rs` — Hand-written. Structured format parsers (pwdump, shadow, DCC) using regex capture groups. Each format has a regex + extractor function returning `FormatMatch`.
- `src/filter.rs` — Rarity range filtering and tag include/exclude logic.
- `src/input.rs` — Input resolution: raw text, file reading (line-by-line), or recursive directory walking.
- `src/output.rs` — Pretty terminal output (colored by rarity) and JSON serialization.

### Adding a New Identifier

1. Create an extractor module or add to an existing one
2. Add a struct implementing `Identify` in `src/identifier.rs`
3. Add it to `all_identifiers()` (order matters — higher confidence first)
4. Add `mod <name>;` in `main.rs` if it's a new module

### Adding a New Format (structured hash dump)

Add to `src/formats.rs`: define a `LazyLock<Regex>`, write an `extract_*` function returning `Vec<FormatMatch>`, and add to the `FORMATS` array.
